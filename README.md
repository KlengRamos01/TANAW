# TANAW
A web and mobile app that gives Filipino clear, location-specific weather forecasts for the days ahead, telling them which destinations to avoid due to storms or unsafe conditions, and recommending nearby places with better weather instead.

Travelers and residents across the Philippines lack a single tool that turns multi-day weather forecasts into clear, location-specific guidance and flagging which destinations to avoid due to storms or unsafe conditions and which nearby places have better weather instead, a gap felt daily by tourists, commuters, and event planners, especially during typhoon season. Today, they cobble together answers from scattered sources like PAGASA bulletins, generic weather apps, and social media storm updates, a slow and fragmented process that often delivers warnings too late and offers no actionable alternatives.

Core Features

1.	7-Day Destination Search with Plain-Language Daily Breakdown User searches a specific named destination (e.g., "El Nido," not just "Palawan province") and sees a day-by-day card for the next 7 days, each showing one risk-relevant sentence generated from PAGASA (https://bagong.pagasa.dost.gov.ph/automated-weather-station) and model data from Open Weather API and Gemini API instead of OpenAI (e.g., "Friday: Heavy rain expected, rough seas, ferry crossings likely affected"), not raw meteorological data. 
2.	Three-Tier Destination Risk Badge (Red / Yellow / Green) Each searched destination displays a single color-coded badge for the user's selected date range, calculated from a defined ruleset (rainfall thresholds, storm signal number, wind speed, sea condition) rather than a generic "% chance of rain." 
3.	Alternative Destination Suggestions (Same Region, Better Forecast) When a destination badge is Red or Yellow, the app shows exactly 3 alternative destinations within the same island group (Luzon, Visayas, or Mindanao) that have a Green badge for the same date range, ranked by proximity to the original search.
4.	Saved Trip Alerts with Risk-Change Notifications User saves a trip by entering a destination and a specific date range (not an open-ended "watch this place"). The app sends a push notification only when the risk badge for that saved trip changes tier (e.g., Green to Yellow) at any point before the trip start date. 
5.	Data Source and Timestamp Disclosure on Every Forecast Every forecast card and risk badge displays the specific source (e.g., "PAGASA Severe Weather Bulletin #12") and a last-updated timestamp, plus a persistent one-line disclaimer that the app interprets official forecasts and is not an independent weather authority.
